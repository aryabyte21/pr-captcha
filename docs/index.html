Built for maintainer trust, not AI detection theater.

The product is deliberately narrow. It proves a human GitHub user was present for one PR commit before the queue spends maintainer attention. Greptile's OpenClaw study showed what happens when PR volume starts behaving like email spam. pr-captcha is the cheap knock-before-entry check for that world.

Receipt contract

Every pass is tied to a repository, pull request, GitHub account, Turnstile solve, and exact head SHA.

Identity Solver is tied to a GitHub account. read:user
Presence Browser challenge was verified on the Worker. Turnstile
Scope Receipt applies only to the current head commit. head.sha

Clear boundaries

pr-captcha is not a code-quality judge, bot detector, or maintainer replacement.

No patch checkout The Worker never executes untrusted contributor code.
No model guessing It does not claim to know how a patch was written.
No permanent pass A pushed update invalidates the old receipt.

Set it up in four steps.

Use the hosted Worker first. Self-hosting can wait until you have a reason to own the infrastructure.

Install app Make policy
01
Install the hosted GitHub App Pick the repositories you want to protect. This is the normal free path.
02
Generate a simple policy Use the wizard to create .github/pr-captcha.yml. Start with fork PRs and first-time contributors.
03
Test one harmless PR Open a disposable PR and confirm the check appears, the link opens, and the exact commit turns green after verification.
04
Require the check when ready Add pr-captcha/human to required checks. Rollback is removing that single required check.
